# 分页接口使用文档

## 通用分页请求参数

所有分页接口都使用以下通用参数结构：

```json
{
  "current": 1,
  "pageSize": 10,
  "sortField": "createTime",
  "sortOrder": "desc"
}
```

**参数说明：**
- `current` (int, 必填): 当前页码，从1开始
- `pageSize` (int, 必填): 每页大小，**最大限制20条**
- `sortField` (string, 可选): 排序字段
- `sortOrder` (string, 可选): 排序方式，`asc`(升序) 或 `desc`(降序)，默认升序

## 通用分页响应格式

```json
{
  "code": 0,
  "data": {
    "records": [],
    "total": 100,
    "size": 10,
    "current": 1,
    "pages": 10
  },
  "message": "ok"
}
```

**响应字段说明：**
- `records`: 当前页数据列表
- `total`: 总记录数
- `size`: 每页大小
- `current`: 当前页码
- `pages`: 总页数

## 分页接口列表

### 1. 分页获取用户列表（管理员）
- **接口地址**: `POST /user/list/page`
- **权限要求**: 管理员
- **扩展参数**: `userName`(用户昵称), `userRole`(用户角色)

### 2. 分页获取用户脱敏信息列表
- **接口地址**: `POST /user/list/page/vo`
- **权限要求**: 无
- **特点**: 返回脱敏的用户信息

#

- **每页最大20条记录**，超过会返回参数错误
- 管理员接口需要admin角色权限
- 个人数据接口需要登录状态
- 公开接口返回脱敏数据
- 对于搜索功能，优先使用ES搜索接口 `/post/search/page/vo`
